# Tema 4. Trabajo con API

## Unidad 3. Peticiones PUT, DELETE y PATCH con Fetch

## Ejercicio 4.3.2 (Castellano)

A partir de la clase `ReqRes` implementada en el ejercicio anterior vamos a ampliarla con nuevos métodos que nos permitan practicar el trabajo con `DELETE` mediante `fetch` sobre una API.

Recuerda que trabajamos con el servicio web gratuito proporcionado por:<br>
https://reqres.in

Cuya documentación y entorno de pruebas puede consultarse aquí:<br>
https://reqres.in/api-docs/

Para ello:


### A. Preparación del fichero `u3e2.js`

Copia el código correspondiente a la clase `ReqRes` (y su exportación) del ejercicio del Tema 4 ubicado en `/u3/u3e1.js` de manera que podamos partir de esa implementación.


### B. Método `deleteUser`

Añade el método `deleteUser` que recibe como parámetros:

- `id`: id del usuario a actualizar

Este método realizará las siguientes tareas:

1) Si el parámetro `id` es vacío o no es un número entero positivo, devuelve la promesa resuelta dando como respuesta el siguiente objeto:

```js
{
    error: 'User id is missing or is not valid'
}
```

2) Si el `id` se recibe correctamente, realizará la petición `fetch` por medio de `DELETE` al **endpoint** `ENDPOINT_USER` sustituyendo el `{id}` por el parámetro `id` recibido, y da como respuesta la promesa que devuelve la propia petición `fetch`.
